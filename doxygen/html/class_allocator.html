Detailed Description

Resides within the application runtime and tracks all memory allocations and frees, ensuring data is not corrupt or the memory leaked.

This overrides all new/delete/malloc/free calls so they will NOT be usable, ensuring the correct functions are always called for the application. As system headers should always be included before this one, it will not stop normal API functions from working - though no doubt we'll encounter a problem along the way, probably via inclusion order.
